# Cloudflare Pages 配置指南

## 项目信息

- **GitHub仓库**: https://github.com/foypeng/iso9001-toolkit
- **项目类型**: 静态网站 (HTML/CSS/JS)
- **公开目录**: `public/`
- **生产分支**: `master` (建议)

## 需要的Token和信息

### 1. Cloudflare账号信息

**必需**:
- Cloudflare账号邮箱
- Cloudflare账号密码

**可选**:
- Cloudflare API Token (用于自动化配置)

### 2. GitHub访问权限

**必需**:
- GitHub账号访问权限
- 仓库访问权限 (foypeng/iso9001-toolkit)

**可选**:
- GitHub Personal Access Token (如果需要私有仓库访问)

## 详细配置步骤

### 步骤1: 登录Cloudflare

1. 访问 https://dash.cloudflare.com
2. 输入你的Cloudflare账号邮箱和密码
3. 完成登录

### 步骤2: 创建Pages项目

1. 在左侧菜单中找到 **Pages**
2. 点击 **Create a project**
3. 选择 **Connect to Git**

### 步骤3: 授权GitHub

1. 点击 **Connect GitHub** 按钮
2. 浏览器会跳转到GitHub授权页面
3. 如果你已登录GitHub，会显示授权请求
4. 点击 **Authorize Cloudflare** 授权访问
5. 授权完成后，会自动跳转回Cloudflare

### 步骤4: 选择仓库

1. 在仓库列表中找到 **foypeng/iso9001-toolkit**
2. 点击仓库名称旁边的 **Begin setup** 按钮

### 步骤5: 配置构建设置

| 设置项 | 值 | 说明 |
|--------|-----|------|
| **项目名称** | `iso9001-toolkit` | 你的项目名称 |
| **生产分支** | `master` | 主要部署分支 |
| **构建命令** | (留空) | 静态网站不需要构建 |
| **构建输出目录** | `public` | 静态文件所在目录 |

**重要**:
- 构建命令留空，因为这是纯静态网站
- 构建输出目录必须是 `public`
- 生产分支选择 `master` (如果使用其他分支，请相应修改)

### 步骤6: 部署设置

1. 点击 **Save and Deploy** 按钮
2. 等待部署完成 (通常1-2分钟)
3. 部署完成后，你会看到:
   - 部署状态: Success
   - 访问URL: `https://iso9001-toolkit.pages.dev`
   - 部署时间: 显示具体时间

## 自定义域名 (可选)

### 添加自定义域名

1. 在项目设置中找到 **Custom domains**
2. 点击 **Set up a custom domain**
3. 输入你的域名 (例如: `compliancepro.com`)
4. 点击 **Continue**

### 配置DNS记录

根据Cloudflare的提示，你需要添加以下DNS记录:

```
类型: CNAME
名称: www (或你想要的子域名)
目标: iso9001-toolkit.pages.dev
TTL: 自动
```

或者:

```
类型: A
名称: @ (根域名)
IP地址: [Cloudflare提供的IP]
TTL: 自动
```

## 环境变量 (可选)

如果需要配置环境变量:

1. 在项目设置中找到 **Environment variables**
2. 添加变量:
   - 变量名: `NODE_VERSION`
   - 值: `18`
   - 环境: Production

## 常见问题解决

### 问题1: 部署失败

**症状**: 部署状态显示失败
**可能原因**:
- 构建输出目录配置错误
- 分支选择错误
- 文件权限问题

**解决方案**:
1. 检查构建输出目录是否为 `public`
2. 确保选择正确的分支
3. 检查文件权限

### 问题2: 页面样式丢失

**症状**: 页面布局混乱，样式不显示
**可能原因**:
- CSS文件路径错误
- 文件未正确部署

**解决方案**:
1. 检查HTML中的CSS引用路径
2. 确保CSS文件在public目录下
3. 检查文件大小是否正确

### 问题3: 404错误

**症状**: 访问页面显示404
**可能原因**:
- index.html不在根目录
- 路由配置问题

**解决方案**:
1. 确保index.html在public目录根目录
2. 检查文件名大小写
3. 如果是SPA，配置适当的路由规则

### 问题4: 部署后没有更新

**症状**: 推送代码后网站没有更新
**可能原因**:
- 自动部署未触发
- 构建失败
- 缓存问题

**解决方案**:
1. 检查GitHub webhook配置
2. 查看构建日志
3. 清除浏览器缓存
4. 手动触发重新部署
