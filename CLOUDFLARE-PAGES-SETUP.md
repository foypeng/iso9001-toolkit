# Cloudflare Pages GitHub 集成配置指南

## 项目信息

- **GitHub仓库**: https://github.com/foypeng/iso9001-toolkit
- **项目类型**: 静态网站 (HTML/CSS/JS)
- **公开目录**: `public/`
- **生产分支**: `master` (建议)

## 配置步骤

### 第一步：登录Cloudflare

1. 访问 https://dash.cloudflare.com
2. 使用你的Cloudflare账号登录

### 第二步：创建Pages项目

1. 在左侧菜单选择 **Pages**
2. 点击 **Create a project**
3. 选择 **Connect to Git**

### 第三步：授权GitHub访问

1. 点击 **Connect GitHub**
2. 如果需要，授权Cloudflare访问你的GitHub账号
3. 在仓库列表中选择 **foypeng/iso9001-toolkit**

### 第四步：配置构建设置

| 设置项 | 值 | 说明 |
|--------|-----|------|
| **项目名称** | `iso9001-toolkit` | 你喜欢的名称 |
| **生产分支** | `master` | 主要部署分支 |
| **构建命令** | (留空) | 静态网站不需要构建 |
| **构建输出目录** | `public` | 静态文件所在目录 |

### 第五步：部署设置

1. 点击 **Save and Deploy**
2. 等待部署完成 (通常1-2分钟)
3. 部署完成后，你会得到一个 `*.pages.dev` 域名

## 常见问题

### 1. 部署失败

**可能原因**:
- 构建输出目录配置错误
- 分支选择错误

**解决方案**:
- 确保构建输出目录为 `public`
- 确保选择正确的分支

### 2. 页面样式丢失

**可能原因**:
- CSS/JS文件路径错误
- 文件未正确部署

**解决方案**:
- 检查HTML中的资源引用路径
- 确保所有资源文件都在public目录下

### 3. 404错误

**可能原因**:
- index.html不在根目录
- 路由配置问题

**解决方案**:
- 确保index.html在public目录根目录
- 如果是SPA，配置适当的路由规则

## 自动部署

1. 推送代码到GitHub
2. Cloudflare Pages自动检测更改
3. 自动构建和部署
4. 更新生产环境
